A
Before the audit — preparation
1
Open Process Tracker and filter by High priority + Not Started
These are your preparation focus. Identify which process owners need to be briefed and which auditors need to be assigned before the session.
2
Open the Audit Card for each process you will cover
Review the required evidence list, ISO clause, key risk and current gaps. Print or screenshot the card to use in the audit session itself.
3
Check the Action Log — filter by Open + High priority
Any actions from previous audits or SGS observations must be addressed. Confirm with owners that evidence of completion is on SharePoint before the session.
4
Set process status to In Progress
Update the process status as soon as active preparation begins. This keeps the Overview accurate and signals readiness to the Quality Manager.
B
During the audit — live use
5
Use the Audit Card as your live checklist
Work through required evidence row by row with the process owner. Note findings against each item — what is available, what is missing, and any improvements to raise.
6
Log non-conformities and improvements immediately
Add any NC or improvement to the Action Log before leaving the session. Assign a reference number, owner and target date. Do not rely on notes — log in real time.
7
Update process status at the end of the session
If the audit report is signed off and no NCs are outstanding, mark Complete. If NCs remain open, leave as In Progress. Never mark Complete with open actions.
C
After the audit — close out
8
Populate the Audit Overview Report
Enter the conformity score, NC count and improvement count for the completed audit. This is currently missing for 7 completed processes — this must be done before the SGS visit on 11 June.
9
Update the Active Improvement Log on SharePoint
All NCs and improvements must be entered on the Improvement Log with owner, priority and deadline. The Log is currently dated Sep 2023 and needs re-versioning to v4.0.
10
Monitor the Action Log weekly until SGS
Filter by Open and check for overdue items (shown in red). Chase action owners. Close actions only when evidence of completion is confirmed and uploaded to SharePoint.
Status system — apply consistently across all 25 processes
Complete
Audit report signed off, all NCs closed, conformity score entered in Audit Overview Report
In progress
Audit underway, or complete with open NCs or actions not yet closed out
Not started
No audit activity — no date set, no report prepared, no auditor assigned
Complete with open NCs
Audit done and scored but one or more non-conformities remain open on the Action Log
